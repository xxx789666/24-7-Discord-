【discord-lobster】— Discord AI 社群管理機器人骨架
  - 三支腳本：welcome.js / vibes.js / memory.js
  - 零 npm 依賴，$0/月，Node.js + Gemini Flash 驅動
  - 可客製化 prompt 語言與人格風格

【prompt-defense-audit】— LLM 提示詞安全掃描器
  - 純 Regex，<5ms，100% 可重現
  - 掃描 12 種 OWASP 攻擊向量
  - 給出 A–F 評級，確保 prompt 不被注入攻擊

【OpenClaw】— 本地 Windows/WSL2 AI 助理核心引擎
  - 開源、自架，資料不離機
  - 支援 Discord 頻道作為操控介面
  - 支援 cron 排程、記憶、技能擴充
  - WSL2 安裝指令：
    curl -sSL https://openclaw.ai/install.sh | bash

【free-tier-agent-fleet】— 多代理人艦隊框架（新整合）
  - 4 代理人（CEO/Social/Security/Advisor）套用至 Arthur 多角色
  - 每日 ~105 個自動化任務，使用 Gemini 免費額度（1,500 RPD）僅佔 7%
  - 6 層品質架構：Quality Gate / 資料驅動 / 對話串接 / 同儕審查 / 週策略 / 研究鏈
  - Self-Healing 三層自癒系統
  - Evolution 自進化模組（Dream Cycle / Agent Arena / Memory Consolidate）
  - Telegram Commander 遠端控制

🔧 6 層品質架構（整合自 free-tier-agent-fleet）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Layer 1：Quality Gate（每篇推文）
    每次 /SPAWN 發文：生成 → 自評（1-10分）→ 低於 7 重寫
    自動阻擋低品質內容，存入 drafts/ 待人工審核

  Layer 2：資料驅動上下文（0 額外 RPD）
    發文前自動讀取：
    - POST-PERFORMANCE.md（歷史發文表現）
    - COMPETITOR-INTEL.md（競爭對手動態）
    純文字注入，不額外消耗 LLM token

  Layer 3：對話串接（#ask-arthur-agent）
    追蹤每條串的深度（max 2 輪/串），批次限制 5 則/次
    避免觸發 Discord 反垃圾機制

  Layer 4：同儕審查（跨腳本）
    publisher.js 發文前 → arthur-agent 審查行銷宣稱正確性
    arthur-agent 回答前 → 讀取 memory.json 個人化校驗

  Layer 5：週策略（每週日 12:00）
    三個視角（市場/社群/法規）各提出優先項目
    → Arthur 統整成下週內容計畫 → 存入 WEEKLY-STRATEGY.md

  Layer 6：Research Chain（每天 09:00 + 17:00）
    RSS 掃描（0 RPD） → Jina Reader 摘要（0 RPD）
    → Gemini 分析相關性 → 寫入 RESEARCH-NOTES.md
    ⚠️ 每次最多處理 5 篇新 URL（防止 RPD 爆炸）

STEP P-4｜Windows 24/7 持續運行規劃（WSL2 升級版）
──────────────────────────────────
  方案A（推薦）：WSL2 Ubuntu + systemd timers + OpenClaw
    - WSL2 提供完整 Linux 環境，支援 systemd timers（比 PM2 cron 更穩定）
    - OpenClaw 負責 AI 決策、Agent 深度問答、記憶管理
    - systemd timers 取代 PM2 cron，25+ 計時器精確控制
    - /SPAWN 指令透過 OpenClaw Agent 模式接收與處理
    - Self-Healing 每 10 分鐘自動偵測並修復常見故障

  方案B：純 discord-lobster + PM2（輕量備選）
    - 不需要 OpenClaw，完全輕量
    - /SPAWN 改為獨立 Node.js 腳本監聽
    - 缺少 Self-Healing 和 Evolution 模組

  監控機制：
    - logs/ 資料夾記錄所有操作
    - 每天 06:00 Bot 自我健康報告 → #bot-logs
    - Self-Healing Sentinel 每 10 分鐘自動偵測 → 自動修復 → TG 通知
    - TG Commander 支援 /status /restart /leads /post /heal 等遠端指令

STEP P-5｜Self-Healing 三層自癒系統（新增）
──────────────────────────────────
  每 10 分鐘執行：

  Layer 1（Sentinel 哨兵）
    偵測 6 種常見問題 → 執行對應修復腳本：
    - fix-gateway.sh       — 重啟 OpenClaw Gateway
    - fix-gemini-ratelimit.sh — 設定 1 小時冷卻鎖（RPD 用盡時）
    - fix-disk-space.sh    — 清理 log、temp，釋放磁碟
    - fix-failed-timers.sh — 重置失敗的 systemd timer
    - fix-memory-leak.sh   — 重啟記憶體洩漏的程序
    - fix-webhook.sh       — 驗證並重新連接 Discord Webhook

  Layer 2（AI 診斷）
    Layer 1 修復失敗 → 收集錯誤日誌 → 呼叫 Gemini 診斷
    → 取得 AI 建議的修復步驟

    問題描述 + 已嘗試的修復 + AI 建議 → 等待管理員處理

  Predictive Ops（預測性運維）
    每 10 分鐘採集 6 項指標（滾動 144 個樣本 = 24 小時）
    偵測異常趨勢（回應時間增長、磁碟填充預測、錯誤率突增）
    在問題發生前預警

STEP P-6｜Evolution 自進化模組（新增）
──────────────────────────────────
  Closed-Loop Learning（apply-strategy.js）
    分析發文表現 → 計算話題/語言風格權重
    → 自動調整 Arthur 的內容偏好（加強有效的，減少無效的）

  Dream Cycle（每天 off-peak 時段，每日 6 次）
    僅在系統空閒時執行：
    1. 預生成明日物件推文草稿（存入 drafts/）
    2. 分析最近 7 天成員常見問題 → 更新 FAQ bank
    3. Arthur 自我反思與改進筆記

  Agent Arena（每週日）
    對 Arthur 不同回答風格進行競爭排名：
    60% 成員互動率 + 40% 品質評分
    → 勝出風格獲得更多使用機會

  Memory Consolidation（每週日 23:00）
    整合：Arena 評分 + 策略權重 + Dream 反思 + 自癒日誌 + 發文表現
    → Gemini 合成 WISDOM.md → 持久化 Arthur 的跨週智慧
